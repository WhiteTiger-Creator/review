Given the position, report whether the named player has already won and by which shape. /app holds a small Fortran program that reads one instance file and prints a single line. Running make in /app produces /app/hexverdict, and the program is called as /app/hexverdict instance.json. An instance carries the side length of the hexagonal board, the player the question is about, and every stone already on the board, keyed by its axial coordinate pair. The printed line is an object whose win field is true or false and whose type field names the shape that was found, or is null when there is none. Reading the instance, checking it and printing that line are already in place in /app/src/main.f90. What is missing is the routine in /app/src/judge.f90 that settles the verdict, which right now answers no win for every position handed to it.

A player has won once a single connected group of their own stones ties two of the six corners together, or reaches three separate edges of the hexagon, or closes a loop around at least one point. Which points count as corners, which of them count towards an edge, what has to hold for a loop to count as closed, and which single name to report when a position answers to more than one of the three are all set out in /app/docs/rules.md. Follow that document to the letter.

Work inside /app. The command line and the shape of the printed line stay as they are. Anything else about how the program is put together is yours to rework if getting the verdict right calls for it.
