Repair /app/environment to publish the CMake ABI matrix without changing /app/environment/include/termatrix/matrix.h. Use the script at /app/environment/scripts/build_matrix.sh and the schemas and build commands defined in /app/environment/docs/release-contract.md to build the ABI matrix. If the TERMATRIX_TARGETS build variable is not set, build the glibc-x86_64 and musl-x86_64 targets. Otherwise, build each of the TERMATRIX_TARGETS in its requested order. Any targets that are not understood by this repository will fail prior to publishing the ABI matrix. The stale success content in /app/output/abi-matrix/build-cache-provenance.json will be cleared prior to a failed request completing. All previous artifacts will remain untouched.

The output of this action will be the files located at /app/output/abi-matrix/build-cache-provenance.json, /app/output/abi-matrix/artifacts/<target>/cache_provenance.json, /app/output/abi-matrix/artifacts/<target>/interface_ledger.json, /app/output/abi-matrix/artifacts/<target>/lib/pkgconfig/termatrix.pc, /app/output/abi-matrix/artifacts/<target>/lib/cmake/Termatrix/TermatrixConfig.cmake, and /app/output/abi-matrix/artifacts/<target>/lib/cmake/Termatrix/TermatrixTargets.cmake. The cache provenance report and interface ledger use JSON schema version 2 and the documented generator. Each build’s cache units use the schema defined at cache.units. The generated pkg-config and CMake files form a relocatable CMake package. Static or manual outputs are insufficient. Rebuilding will recover corrupted cache entries, clean stale artifacts, regenerate tampered interfaces, and keep relocated pkg-config and find_package(Termatrix CONFIG REQUIRED) consumers building successfully.
