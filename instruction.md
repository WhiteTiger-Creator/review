Build a structured finance waterfall engine at /app/waterfall.go. Reads /app/input/collections.json, writes /app/output/waterfall.json. Input has tranches (id, class A/B/C/residual, balance, coupon_rate) and periods (period, collections, defaults, prepayments, senior_fee, servicer_fee). Original pool balance = sum of starting tranche balances. Run: cd /app && go build -o /app/waterfall . && /app/waterfall

Each period: available = collections - defaults + prepayments - senior_fee - servicer_fee. Loss allocation first: if defaults this period exceed 3% of current pool balance (sum of all tranche balances at period start), write down the excess bottom-up (residual, then C, B, A). Pay interest A then B then C (post-writedown balance * coupon_rate / 12, 2dp). Shortfall capitalizes as PIK: add the shortfall to the tranche's outstanding principal balance each period and track cumulative PIK per tranche. Scheduled principal = collections - total_interest_paid - senior_fee - servicer_fee - prepayments, floored at zero. Distribute min(scheduledPrincipal, available) per mode: sequential pays A first then B then C, pro_rata splits proportionally by balance among A/B/C (round each share 2dp, no redistribution). Then distribute min(prepayments, available) pro-rata across A/B/C by balance. Excess handling: first fill reserve to cap (2% of original pool). After reserve at cap, pay off cumulative PIK in order A then B then C (reduce balance by recovery amount, cap at available and at tranche's cumulative PIK). PIK recovery is reported only in pik_recovered and must NOT be added to principal_paid. Then if OC test passed (pre-distribution), remainder flows as turbo to most senior with balance (A then B then C). No turbo if OC failed. Round available to 2dp after each subtraction.

Mode starts sequential. Evaluate triggers before distributing principal. OC test: (A+B+C balances) < 0.92 * current pool. pool_factor = current pool / original pool. Switches to pro_rata when ALL pass: pool_factor > 0.50, cumulative_default_rate < 0.04, and OC passes. Any breach after switching locks sequential permanently. cumulative_default_rate = running total defaults / original pool. When it first reaches 0.03, tranche C coupon increases by 0.01 permanently (applies that period). Clean-up call: evaluated at the end of each period after all distributions (interest, principal, prepayments, reserve, PIK recovery, turbo), using the sum of all tranche ending balances for that period (A+B+C+residual). If that sum is below 10% of original pool balance and still positive, the clean-up triggers in THAT SAME period: set clean_up=true, pay all remaining tranche balances as principal (zeroing them), and terminate the deal (no further periods). Output: array of {period, tranche_payments [{id, interest_paid, principal_paid, ending_balance, pik_amount, pik_recovered}], reserve_balance, mode, clean_up}.
