Build a deterministic factorization-machine click-through-rate modeling engine under /app on the compiling Rust baseline. Deliver /app/bin/fmctl with train, predict, and evaluate capabilities so advertising scientists can fit a binary classifier with linear and pairwise latent interactions, reuse its canonical model state, and measure held-out log-loss and AUC.

The verifier rebuilds fmctl from Rust sources on every test run. The command surface and file paths are defined in /app/docs/cli-surface.md. Feature identity and collision accumulation follow /app/docs/feature-index.md. Prediction math is specified in /app/docs/fm-score.md, while optimization behavior is split between /app/docs/gradient.md and /app/docs/sgd-schedule.md. Canonical model persistence is defined in /app/docs/model-format.md, and evaluation follows /app/docs/metrics.md.

Training must be deterministic for a fixed seed and input, including example ordering, mini-batch updates, latent initialization, and model serialization. Prediction and evaluation must load the persisted model rather than retraining. When TB3_FM_DATA is an absolute path it overrides the train data path; relative overrides must be ignored. Repeated training with unchanged arguments must produce byte-identical model bytes. The empirical click-rate helper under /app/src/decoy is dashboard-only and is not authoritative for training, prediction, or evaluation.
