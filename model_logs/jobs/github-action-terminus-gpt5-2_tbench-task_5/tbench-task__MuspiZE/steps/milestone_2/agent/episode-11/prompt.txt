7:POST /api/notes accepts JSON with string title and content. Title must be present and non-empty; an empty object or missing or blank title is a client error.
Success is HTTP 201 with a body containing numeric id plus stored title and content. GET /api/notes returns a JSON array of objects with id, title, and content.
 Optional limit and offset query parameters are integers: default limit 50, maximum 100, default offset 0. /api/notes?limit=100 returns at most one hundred rows
; /api/notes?limit=999 is capped the same way. The total row count before paging goes in the X-Total-Count header.
9:GET /api/notes/{id} returns that object. A missing id returns HTTP 404 with JSON {"error":"not found"} using that exact lowercase message string, including fo
r ids such as /api/notes/99999 on GET, PUT, or DELETE. PUT uses the same field rules as POST, returns HTTP 200 with saved fields when the row exists, or the sam
e not-found JSON when it does not. Successful DELETE returns HTTP 200 with JSON {"status":"deleted"}; deleting a missing id returns the same not-found JSON as G
ET. Reject POST or PUT bodies larger than 65536 bytes with HTTP 413 and a JSON error body.
