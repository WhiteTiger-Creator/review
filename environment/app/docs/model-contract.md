# Model Contract

The command line program /app/bin/apnea_hmm executes the duration-aware sleep-stage model. It accepts --train, --adapt, --validation, --infer, and --out-dir arguments. The train and validation directories contain labeled traces. The adapt and infer directories contain unlabeled traces. The output directory must be created if missing. Existing model.json, predictions.csv, posterior.csv, apnea_events.csv, and validation_metrics.json in that directory must be replaced on each run.

Train the supervised duration-aware two-component full-covariance Student-t mixture model described in /app/docs/duration-decoder.md, then adapt it with the fixed anchored procedure in /app/docs/adaptation-policy.md. Use the state order quiet, flow_limited, apnea, and the feature order airflow_flatness, spo2_drop, resp_pause, body_motion. Validation and inference use the final adapted model.

Run duration-aware Viterbi and forward-backward inference in log space for each validation and inference sequence. The decoder's tie, expanded-state, and posterior rules are defined in /app/docs/duration-decoder.md.

Write model.json with exactly states, features, training_sequences, training_rows, adaptation_sequences, adaptation_rows, adaptation_iterations, emission_degrees_of_freedom, adaptation_log_likelihood, start_probability, duration_cap, duration_continue_probability, exit_destination_probability, and emission. emission_degrees_of_freedom is the numeric value 5. adaptation_iterations is the number of completed adaptation rounds. adaptation_log_likelihood is the numeric total calibration log likelihood for each round, in chronological order before that round's re-estimation. start_probability and duration_cap are objects keyed by state. duration_continue_probability is keyed by state and then elapsed age as a decimal string from 1 through that state's cap. exit_destination_probability is keyed by state and then every other state, never itself.

emission is keyed by state. Each state emission has exactly mixture_weight and components. Both are objects with exactly the string keys 0 and 1. mixture_weight values are numeric and sum to one for a state. Each components entry has exactly mean and covariance objects, each keyed by every documented feature. covariance is keyed again by every documented feature, so each component stores one complete numeric covariance matrix.

Write predictions.csv with the header sequence_id,t,predicted_state. Write posterior.csv with the header sequence_id,t,quiet_posterior,flow_limited_posterior,apnea_posterior,entropy. Rows in both files are sorted by sequence_id and then t. Write apnea_events.csv with the header sequence_id,start_t,end_t,length,mean_spo2_drop,max_resp_pause,mean_apnea_posterior,severity,preceding_state. Events are sorted by sequence_id and then start_t. These three CSV files contain only sequences from --infer. Do not write calibration or validation rows, posteriors, or events to them. An apnea event is a maximal run of at least two consecutive Viterbi-predicted apnea rows in one sequence. Severity is high when the run length is at least 3, max_resp_pause is at least 15, or mean_apnea_posterior is at least 0.85; otherwise it is watch. preceding_state is the Viterbi-predicted state immediately before the run, or start if the run begins at the first row.

Validation_metrics.json has numeric accuracy, macro_f1, mean_negative_log_likelihood, and mean_posterior_entropy fields plus a confusion object. confusion is keyed first by true state and then by predicted state: {"confusion":{"quiet":{"quiet":0,"flow_limited":0,"apnea":0},"flow_limited":{"quiet":0,"flow_limited":0,"apnea":0},"apnea":{"quiet":0,"flow_limited":0,"apnea":0}}}. Each value is an integer count. These rows and columns follow documented state order.

Numeric values in JSON and numeric CSV columns must not be quoted strings. Their serialized values must recover to within 1e-9 of the computed value; emit at least 15 significant decimal digits. Scientific notation is allowed.
