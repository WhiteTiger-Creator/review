# Attestation HTTP contract

`GET /up` remains available. `POST /api/v1/attestations` accepts an uncompressed tar in the multipart `archive` field with media type `application/x-tar`; `release_ref` is optional and otherwise uses the configured default. A supplied release must be a full `refs/tags/...` reference that resolves to a GPG-valid annotated tag signed by the configured allowed fingerprint.

Archive structure is validated immediately after the upload is read and before release validation, cache lookup, remote access, LFS hydration, or policy scanning. Unsafe archive structure returns HTTP 422 even when the configured corpus cannot be resolved. Corpus remotes may be read-only and owned by a different local account than the Puma process. Cache reuse is valid only for a completely verified release under the same remote URL, allowed signer fingerprint, and release reference; changing any trust input requires fresh verification.

A successful request returns HTTP 200 with compact JSON containing `schema_version`, `archive_sha256`, `baseline`, `verdict`, `findings`, and `signature`. `baseline` contains `ref`, `tag_object`, `commit`, `signer_fingerprint`, and `policy_sha256`; each finding contains `path`, `kind`, `rule_id`, `severity`, `line`, and `evidence_sha256`; `signature` contains `alg`, `key_id`, and base64url `value`. Invalid references and unsafe archives return HTTP 422; absolute or parent-traversing archive paths use error code `unsafe_path`. Tag verification, LFS hydration, or policy availability failures return HTTP 424. Errors use `{"error":{"code":...,"message":...}}`. Successful bodies are deterministic and must not include matched secret or private-key bytes.
