# Spherical Harmonic Flux Contract

## Inputs

The input directory contains four simple comma-separated files with exact headers. This is not RFC-style CSV: fields are split literally on commas, quoting is not supported, and any double quote character (`"`) anywhere in an input field is malformed, even a field such as `"bad"` with no embedded comma. Token fields such as ids, ring ids, regions, and mode kinds must be nonempty and must not contain commas, newlines, carriage returns, or double quotes. All decimal fields must parse as finite double-precision numbers; `NaN`, positive infinity, and negative infinity are malformed. `settings.csv` is `key,value` and must contain exactly one row for each key `quadrature_order`, `rotation_degrees`, `clip_floor`, and `alert_flux`; quadrature order is an integer from 4 through 24. `rings.csv` is `ring_id,theta_min_deg,theta_max_deg`; theta is colatitude in degrees with `0 <= min < max <= 180`, ring ids are unique, and file order is the ring order. `cells.csv` is `cell_id,ring_id,phi_start_deg,phi_end_deg,region,exposure`; ids are unique, referenced rings must exist, exposure is positive, and `0 < phi_end_deg - phi_start_deg <= 360`. Longitude intervals sweep east from start to end; normalize the start modulo 360, split wrapped intervals at 360, and treat an exact 360-degree width as the full circle. An exact full-circle cell is represented as one longitude segment `[0, 2*pi]` in radians regardless of `phi_start_deg`; do not represent it as `[normalized_start, normalized_start + 2*pi]`. `modes.csv` is `mode_id,ell,m,kind,coefficient`; ids are unique, `0 <= m <= ell`, `kind` is `C` or `S`, `S` rows require `m > 0`, and file order is the mode order.

## Quadrature And Field Evaluation

For quadrature, use tensor-product Gauss-Legendre nodes on each cell's `mu = cos(theta)` interval and each normalized longitude segment, using the requested order in both directions. A longitude rotation is applied only while evaluating the field: `phi_eval = phi + rotation_degrees*pi/180`. The real basis for a mode is `P_ell^m(mu)*cos(m*phi_eval)` for kind `C`, or `P_ell^m(mu)*sin(m*phi_eval)` for kind `S`. Use the unnormalized associated Legendre recurrence with the Condon-Shortley sign: `P_m^m(x)=(-1)^m(2m-1)!!(1-x^2)^(m/2)`, `P_{m+1}^m(x)=x(2m+1)P_m^m(x)`, and `P_l^m(x)=((2l-1)xP_{l-1}^m(x)-(l+m-1)P_{l-2}^m(x))/(l-m)`. The scalar field is the sum of `coefficient*basis` over all modes. Cell area is the unweighted spherical area integral; raw and clipped fluxes are multiplied by the cell exposure. Clipped flux uses `max(field, clip_floor)` at quadrature nodes. For surface-gradient energy, use `|grad f|^2 = (1-mu^2)*(df/dmu)^2 + (df/dphi)^2/(1-mu^2)` at quadrature nodes. The derivative of `P_ell^m` is `(ell*mu*P_ell^m(mu) - (ell+m)*P_{ell-1}^m(mu))/(mu^2-1)`, with `P_{ell-1}^m=0` when `ell=m`; Gauss-Legendre nodes never lie exactly on the pole. All intermediate math uses double precision; round only when writing files.

## Formatting

Write exactly eight UTF-8 reports, each ending with one trailing newline. Decimal fields use exactly six digits; values whose absolute rounded magnitude is below `0.0000005` must be written as `0.000000`.

`cell_flux.csv`: header `cell_id,ring_id,region,area,raw_flux,clipped_flux,mean_raw,mean_clipped,clipped_nodes,dominant_mode`. Rows follow cell file order. `mean_raw=raw_flux/(exposure*area)` and `mean_clipped=clipped_flux/(exposure*area)`. `clipped_nodes` counts tensor quadrature nodes whose raw field is below `clip_floor`, counting every longitude segment separately. `dominant_mode` is the mode id with the largest absolute exposed raw contribution in that cell, breaking ties by earlier mode order.

`ring_summary.csv`: header `ring_id,cell_count,total_area,total_raw_flux,total_clipped_flux,clip_delta,max_mean_cell,alert_count,regions`. Rows follow ring file order. `clip_delta=total_clipped_flux-total_raw_flux`; `max_mean_cell` is the cell id in that ring with largest unrounded `mean_clipped`, breaking ties by earlier cell order; `alert_count` counts cells with unrounded `clipped_flux >= alert_flux`; `regions` is a semicolon-separated `region:count` list sorted lexicographically, or empty when no cells use the ring.

`mode_coupling.csv`: header `region,mode_a,mode_b,coupling,weighted_coupling,correlation`. For every region that appears in `cells.csv`, emit every mode pair with `mode_a` at or before `mode_b` in mode file order. Regions are sorted lexicographically. `coupling` integrates `basis_a*basis_b` over cells in that region without exposure. `weighted_coupling` uses `exposure*basis_a*basis_b`. `correlation` is `weighted_coupling/sqrt(self_a*self_b)` using weighted self-couplings for that same region; emit `0.000000` if the denominator is zero.

`region_balance.json`: compact single-line JSON with top-level key order `total_area`, `total_raw_flux`, `total_clipped_flux`, `clip_delta`, `regions`. Regions are sorted lexicographically and each object has exact keys `region`, `cell_count`, `area`, `raw_flux`, `clipped_flux`, `clip_delta`, `dominant_mode`. The regional dominant mode is based on the largest absolute sum of exposed raw mode contributions in that region, breaking ties by earlier mode order.

`gradient_audit.csv`: header `cell_id,region,theta_energy,phi_energy,total_gradient_energy,anisotropy,dominant_gradient_mode`. Rows follow cell file order. `theta_energy` is `exposure * integral((1-mu^2)*(df/dmu)^2)`, `phi_energy` is `exposure * integral((df/dphi)^2/(1-mu^2))`, and `total_gradient_energy` is their sum. `anisotropy=phi_energy/total_gradient_energy`, or `0.000000` when total gradient energy is zero. `dominant_gradient_mode` is the mode id with the largest exposed self-gradient energy contribution in that cell, where each mode is evaluated alone with its coefficient; ties use earlier mode order.

`latitude_frontier.csv`: header `face,theta_deg,area_left,raw_flux_left,clipped_flux_left,clip_delta_left,active_cells`. Use the sorted unique set of all ring theta boundaries. Face numbering starts at 0. For a face at theta `T`, integrate only the northern portion of each cell with `theta_min_deg < T`, clipping its theta interval to `[theta_min_deg, min(T, theta_max_deg)]`; longitude handling, rotation, exposure, clipping, and quadrature are otherwise identical to full-cell integration. `active_cells` counts cells with positive area included in that prefix.

`mode_spectrum.csv`: header `mode_id,total_raw_contribution,positive_cells,negative_cells,dominant_cell,gradient_energy,gradient_share`. Rows follow mode file order. `total_raw_contribution` is the sum of that mode's exposed raw contribution over all cells, using the same per-cell contribution used for `dominant_mode`. `positive_cells` and `negative_cells` count cells whose unrounded per-cell contribution for that mode is greater than zero or less than zero; exact zero counts as neither. `dominant_cell` is the cell id with the largest absolute unrounded contribution for that mode, breaking ties by earlier cell file order. `gradient_energy` is the sum of that mode's exposed self-gradient energy over all cells, using the same per-mode quantity used for `dominant_gradient_mode`. `gradient_share=gradient_energy/sum(all mode gradient_energy)`, or `0.000000` when the denominator is zero.

`ring_mode_breakdown.csv`: header `ring_id,mode_id,raw_contribution,share_of_ring_raw,gradient_energy,dominant_region`. Rows are ordered by ring file order and then mode file order. For a ring/mode pair, `raw_contribution` is the sum of that mode's exposed raw contribution over cells in that ring, `share_of_ring_raw=raw_contribution/total_raw_flux_for_that_ring`, or `0.000000` when the unrounded ring total raw flux is zero, and `gradient_energy` is the sum of that mode's exposed self-gradient energy over cells in that ring. `dominant_region` is the region in that ring with the largest absolute summed raw contribution for that mode, breaking ties lexicographically by region name; it is empty when the ring has no cells.
